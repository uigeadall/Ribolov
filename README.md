# Ribolov

Expo / React Native приложение за риболов — дневник, карта, видове, лента и профил.

## Локален старт (след `git clone`)

**Необходимо:** Node.js **20+** (препоръка LTS), **npm**. За реално устройство/симулатор с native код — **Xcode** (iOS, само macOS) и/или **Android Studio** (Android).

В терминала, в папката където е `package.json` на проекта:

```bash
npm install
npm run start
```

Ако нямат глобално инсталиран Expo CLI — не им е нужно: командите минават през `npm run` или `npx expo ...`.

### Важно: Expo Go често няма да работи

Проектът включва **native модули** (Firebase за native и др.). **Expo Go** поддържа само фиксиран набор от native функции — при сканиране на QR кода приложението може да **не тръгне**, да покаже **несъвместимост** или да крашне след старт.

Надежден начин за преглед на телефон или симулатор:

```bash
# iOS — нужен Mac + Xcode
npx expo run:ios

# Android — емулатор или телефон с USB debugging
npx expo run:android
```

Първото стартиране компилира native частта и отнема време.

### По-постига се само Metro (без Expo Go)

```bash
npm run start:metro
# после опции за Android emulator / симулатор от менюто на Expo CLI
```

### Чести грешки


| Какво виждат                      | Какво да пробват                                                                    |
| --------------------------------- | ----------------------------------------------------------------------------------- |
| `expo: command not found`         | Ползват `npm run start` или `npx expo start`, не „голям“ `expo` без `npx`           |
| Телефонът не се свързва към Metro | Същата Wi‑Fi мрежа като компютъра, или `npm run start:share` (**tunnel**, по-бавно) |
| Грешки при `npm install`          | Обновен Node 20+, изтриват `node_modules` и пускат `npm install` отново             |
| Не са в правилната папка          | След clone да влязат в директорията, където е този `README` и `package.json`        |


Firebase конфиг файловете `GoogleService-Info.plist` и `google-services.json` са част от репото (ако липсват при pull — нещо не е наред с clone/fork).

## Карта: язовири / реки не се появяват

Данните за водоемите са **в кода** (`src/data/dams.ts`, `src/data/rivers.ts`), не се теглят от облак. Ако при clone всичко е наред, типичните причини са:

1. **Изключени филтри** на екрана „Карта“ — бутоните **„Язовири“** и **„Реки“** трябва да са активни (цветни).
2. **Твърде малък мащаб** — имената се показват след приближаване; маркерите (кръгчета с иконка) са около България при начален изглед — провери дали картата не е изместена другаде.
3. **Android + нативна карта** — проектът ползва `react-native-maps` с Google Maps. В `app.json` е зададен `android.config.googleMaps.apiKey`. В [Google Cloud Console](https://console.cloud.google.com/) за същия проект трябва да е **включен Maps SDK for Android** за този ключ. След промяна на конфигурацията: отново **`npx expo run:android`** (при нужда `npx expo prebuild --clean` преди това).
4. **Обходен вариант без Google Maps** — в `src/config/mapEngine.ts` сложи `USE_REACT_NATIVE_MAPS = false`, за да се ползва Leaflet/OpenStreetMap през WebView (нужен е интернет към OSM плочките).

Ако след това пак има грешка — изпратете **точния текст от терминала** (или екранна снимка); без него не се познава дали е Node, мрежа, Expo Go или native build.
